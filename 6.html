Listener-Konzept

18.2

In einer Listener-Methode definieren wir, was bei einem bestimmten Event (z. B. Buttonklick) passieren soll.

class HauptFenster(QWidget):
    def __init__(self):
        # ...
        self.button.clicked.connect(self.on_button_click)
        # ...
Für einzelne Widget-typische Events lassen sich Listener-Methoden verbinden. Für Buttons gibt es z. B. das Event clicked, welches die Methode, die wir mittels connect übergeben, immer dann aufruft, wenn der Button geklickt wird. Wie jede Methode in Python haben Listener-Methoden als ersten Parameter self. Darüber können wir auf Member-Variablen (im Beispiel: self.result_label) und andere Methoden zugreifen. Anderen Listener-Methoden können wir optional noch weitere Parameter geben, z. B. beim textChanged-Event einer QLineEdit: def text_geaendert(self, text). Somit kommen wir direkt an den Inhalt des Textfelds als String, ohne über die text()-Methode darauf zugreifen zu müssen.
