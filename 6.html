Da Python-Anwendungen plattformübergreifend sind, wurde stets angestrebt, dass GUI-Anwendungen zum Erscheinungsbild des verwendeten Betriebssystems passen. Läuft ein Programm also auf MacOS, sieht beispielsweise ein Button anders aus als unter Windows.

PyQT

Installation von PyQT:

pip install pyqt6       # Windows
pip3 install pyqt6      # MacOS, Linux

PyQT-Klassen:

  • QApplication - Repräsentiert die GUI-Anwendung
  • QWidget - Fenster oder Anwendungskomponente
  • QLabel - Text, der angezeigt werden soll
  • QLineEdit - Text-Eingabefeld
  • QPushButton - Button, den man klicken kann
  • ...

QApplication und QWidget

Im Hauptteil des Programms (unten) wird eine QApplication angelegt und mit exec gestartet. Man übergibt ihr üblicherweise die Konsolenparameter (sys.argv) und wenn sich die Applikation beendet, liefert sie einen Errorcode zurück (0 kein Fehler, >0 Fehler), den man dann an sys.exit weitergibt. Für das Programmfenster legen wir eine Klasse Hauptfenster an, die von QWidget erbt. Im Konstruktor wird der Konstruktor der Superklasse QWidget aufgerufen und der Titel, die Größe, das Aussehen und der Inhalt des Fensters festgelegt. Ein Aufruf der show-Methode zeigt das Fenster an.
